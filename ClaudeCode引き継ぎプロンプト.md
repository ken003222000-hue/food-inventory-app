# 備蓄在庫管理アプリ 引き継ぎプロンプト（Claude Code用）

## プロジェクト概要
家庭の備蓄食料・日用品・薬などを管理するスマホ向けWebアプリを開発中。
AndroidとiPhoneの2台で同じデータを共有・編集できることが必須条件。

---

## 現在の完成物

### ファイル
- `備蓄在庫管理.html`（単一HTMLファイル、依存なし）

### 実装済み機能
1. **在庫一覧表示**
   - 賞味期限でカラーコード表示（期限切れ赤／30日以内オレンジ／3ヶ月以内黄／6ヶ月以内緑／余裕あり青／期限なし灰）
   - 各アイテムに「あと○日」「○日超過」を表示

2. **フィルター・ソート**
   - ステータス別フィルター（ピルボタン）
   - 場所別フィルター（プルダウン）
   - 商品名・場所でキーワード検索
   - 期限近い順／遠い順／商品名順／場所順でソート

3. **追加・編集・削除**
   - 項目：商品名・賞味期限・収納場所・個数・保存状態・備考
   - 削除時に確認ダイアログあり

4. **バーコードスキャン機能**
   - ライブラリ：`html5-qrcode@2.3.8`（CDN: jsdelivr）
   - カメラでJANコードをスキャン
   - Open Food Facts API（`world.openfoodfacts.org`）で商品名を自動取得
   - 商品名が見つからない場合は手動入力

5. **データ保存**
   - localStorage（端末ごとの自動保存）
   - JSONBin.io APIを使ったクラウド同期（オプション、⚙️ボタンから設定）
   - 20秒ごとに自動でクラウドからpull
   - 追加・編集・削除のたびにcloudPush

6. **UI/UX**
   - スマホ最適化（safe-area対応）
   - iOS/Androidホーム画面追加対応（apple-mobile-web-app-capable）
   - トースト通知（保存完了・削除など）

### データ構造
```json
{
  "id": "string",
  "name": "商品名",
  "type": "保存状態（パウチ・缶詰・袋など）",
  "expiry": "YY/MM/DD形式",
  "qty": "個数（文字列）",
  "location": "収納場所",
  "note": "備考"
}
```

### 賞味期限フォーマット
現在 `YY/MM/DD`（例：`27/03/31`）のみ対応。
元データには `2027年3月24日`、`2028.01`、`27/3/1` など表記ゆれがあったが、
現在は全て `YY/MM/DD` に統一されている。
